2022 年 11 月 30 日,OpenAI 发布了 ChatGPT。两个月内,它的用户突破 1 亿--这是历史上增长最快的消费级应用。

很多人第一次发现:计算机竟然能像人一样对话、写文章、写代码。但大部分人不知道的是,这背后是一段跨越 80 年的旅程--从 1943 年第一个数学神经元模型,到 2017 年 Transformer 架构的诞生,再到 2022 年的 ChatGPT。

这份教程就是这段旅程的地图。我们从最基础的"什么是神经网络"开始,一步步走到"如何构建一个完整的 AI Agent 系统"。每一站都建立在前一站的基础上,不跳步,不假设你已经知道什么。

准备好了吗？让我们从第一个问题开始：机器怎么“学习”？

🧵 全景图：Agent 版“从输入 URL 到页面渲染”

💡 先看全貌，再逐章深入
互联网面试有一道经典题：“从输入 URL 到页面加载经历了什么？”这一个问题串起了 DNS、TCP/IP、HTTP、浏览器渲染等整个 Web 技术栈。

我们的“那道题”是：“用户在飞书群里发了一句‘帮我分析一下最近 AI Agent 框架的趋势，写一份 500 字的报告’——从这条消息发出到用户收到回复，经历了哪些环节？”

读完全文后再回来看这张图，你会发现每一步你都完全理解了。 
① 消息的旅程（从用户到 Agent）

  用户在飞书点击发送
    │
    ├─ Ch15 Channel  飞书 Webhook 接收消息
    ├─ Ch14 Gateway  消息进入 OpenClaw 神经中枢
    ├─ Ch15 Binding  匹配规则：这条消息该给哪个 Agent？
    ├─ Ch16 路由    Agent 隔离：加载 workspace（SOUL.md 等）
    └─ Ch13 Session 找到/创建会话，加载历史消息

② 大脑启动（Prompt 组装 → 模型推理）

  Ch19 Prompt Pipeline 组装完整的 messages：
    ├─ Ch04 System Prompt 注入（角色 + 规则）
    ├─ Ch17 Context Engine 精选历史消息（token 预算）
    ├─ Ch18 Skills 目录注入（按需加载）
    ├─ Ch03 工具列表注入（MCP Tools + 本地工具）
    └─ Ch12 Context Window 检查（总 token 不能超限）
        │
        ▼
  消息被 Tokenize 后送入模型 ← Ch0B
    ├─ Ch0C Transformer 内部：Q/K/V Attention 计算
    ├─ Ch0C 位置编码（RoPE）标记词序
    ├─ Ch12 KV Cache 加速推理
    ├─ Ch0D Decoder-only 架构逐 token 生成
    └─ Ch10 SSE 流式返回 token

③ Agent 思考与行动（ReAct 循环）

  Ch01 LLM 变成 Agent：不只是回答，而是行动
        │
  Ch02 ReAct 循环启动 ─────────────────┐
        │                                  │
  Turn 1:                                  │
    🧠 思考：需要搜索最新 Agent 趋势       │
    🔧 Ch03 Tool Calling：生成 JSON      │
    🔧 Runtime 执行 web_search             │
    👁 观察：搜索结果返回                  │
        │                                  │
  Turn 2:                                  │
    🧠 思考：信息不够，再搜一次            │
    🔧 Ch05 RAG 检索：向量搜索记忆库     │
    👁 观察：更多结果                       │
        │                                  │
  Turn 3:                                  │
    🧠 Ch06 Planning：够了，开始写报告  │
    🔧 组织结构，生成 500 字报告          │
    ✅ 完成 ←─────────────────────┘
        │
  如果任务更复杂→ Ch07 委派子 Agent（ACP）
  如果生成内容有风险→ Ch08 幻觉检查

④ 回复的旅程（从 Agent 到用户）

    Ch05 Memory：关键信息写入长期记忆
    Ch18 Skill 控制输出格式
    Ch14 Gateway → Ch15 Channel → 飞书消息送达 ✅

⑤ 幕后支撑（离线训练，不在请求链路上）

  这个 LLM 为什么这么聪明？
    Ch0A 神经网络基础：权重 → 损失函数 → 梯度下降
    Ch0D 三阶段训练：Pre-training → SFT → RLHF
    Ch10b 对齐算法：DPO / GRPO
  想自己实现一个？
    Ch09 手写 60 行 Agent + 30 行 RAG
